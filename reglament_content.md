# Регламент разработки системы аренды жилья

## 1. Цель проекта:
Создать систему управления арендой жилья с базой данных, включающей таблицы арендаторов, объектов недвижимости и информации об аренде.

## 2. Структура базы данных:
- Таблица "tenants" (арендаторы): tenant_id, last_name
- Таблица "properties" (объекты недвижимости): property_id, type, monthly_rent
- Таблица "leases" (аренда): lease_id, tenant_id, property_id, start_date, duration_months

## 3. Функциональные требования:
- Добавление и удаление записей через HTML-формы
- 3 отчетные страницы с запросами
- Пользовательский интерфейс для управления данными

## 4. Технические требования:
- Использование PHP, SQL, HTML, JavaScript
- База данных: MySQL
- Сервер: Open Server Panel
- Инструменты: phpMyAdmin

## 5. Алгоритм решения задачи:

### Описание алгоритма:
- Создание структуры базы данных с тремя таблицами: арендаторы, объекты недвижимости и аренда
- Разработка модели MVC для обработки данных
- Создание интерфейса для добавления, просмотра и удаления записей
- Реализация отчетных функций согласно требованиям
- Внедрение функций обновления данных

### Преимущества алгоритма:
- Модульная архитектура (MVC) позволяет легко поддерживать и расширять систему
- Четкое разделение ответственности между слоями приложения
- Использование подготовленных выражений для предотвращения SQL-инъекций
- Понятный пользовательский интерфейс для управления данными
- Все функции отчетности интегрированы в одну систему

### Методы реализации:
- Использование PHP для серверной логики
- MySQL для хранения данных
- HTML/CSS для пользовательского интерфейса
- JavaScript для клиентской валидации (при необходимости)

## 6. Основные проблемы, на которые следует обратить внимание:

### Безопасность данных:
- Защита от SQL-инъекций через подготовленные выражения
- Валидация вводимых данных
- Защита от XSS-атак через экранирование вывода

### Целостность данных:
- Использование внешних ключей для обеспечения ссылочной целостности
- Проверка корректности вводимых данных (например, дат, числовых значений)
- Обработка ошибок при операциях с базой данных

### Производительность:
- Оптимизация SQL-запросов
- Использование индексов для ускорения поиска
- Эффективное управление соединениями с базой данных

### Пользовательский интерфейс:
- Создание интуитивно понятного интерфейса
- Обеспечение удобства работы с большими объемами данных
- Предоставление возможности фильтрации и сортировки

## 7. Лучшее решение задачи:

### Структура проекта:
- public/: публичные файлы (index.php, CSS, JS)
- src/: исходный код (models.php, controllers.php, views/)
- sql/: SQL-скрипты для создания и заполнения базы данных

### Архитектура:
- Использование шаблона MVC
- Разделение на слои: представление, контроллеры, модель
- Единая точка подключения к базе данных

### Функциональность:
- Полный CRUD для всех сущностей
- 10 различных отчетов согласно требованиям
- Возможность обновления арендной платы

### Безопасность:
- Подготовленные выражения для всех запросов
- Экранирование вывода для предотвращения XSS
- Проверка и валидация входных данных

### Удобство использования:
- Чистый и понятный интерфейс
- Навигация между разделами
- Подтверждение удаления записей
